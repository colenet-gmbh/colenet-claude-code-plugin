# cape — Roadmap

Goal-oriented and deliberately short. The concrete work — every feature and issue — lives
on the board under `docs/work/`; this roadmap only says what each stage is *for* and how we
know it is reached.

## Goal 1 — Someone else can use cape in their own project

One internally consistent version that installs from the marketplace, sets up cleanly in a
fresh repo, and carries the value worth having — the good ideas taken up from the harnesses
we learned from. **Reached when** a colleague can adopt cape in their own project and get
value without hand-holding.

## Goal 2 — A properly good solo-developer harness

cape becomes a harness a single developer reaches for because it makes the everyday work
better — clearer rules, depth where it pays, the guardrail mechanics, and the means to
improve the harness from real use. **Reached when** a solo developer prefers working with
cape to working without it.

## Goal 3 — cape for teams

How several people work well on one product together — the team story beyond the solo,
single-repo focus. **Reached when** a team adopts cape as its shared way of working with AI.
