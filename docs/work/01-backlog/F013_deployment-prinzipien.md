# cape bietet Deployment-Prinzipien

## Outcome

cape sagt *etwas* Nützliches zum Deployment — als **Prinzipien**, nicht als konkrete
Pipeline, weil die konkrete Pipeline stark projektspezifisch ist.

## Open points

- Der Scope ist vorerst bewusst nur Prinzipien; eine konkrete Pipeline bleibt außerhalb von
  cape.
