# Glossary

The **environment glossary** — terms of the surroundings: arc42 conventions, tooling, and
file details that help when writing or reading these docs. It is **not** the project's
domain: the ubiquitous language lives in the **domain glossary**
([chapter 8](08_crosscutting-concepts.md)).

In this repo cape *is* the domain, so the harness concepts (skill, bucket, feature, issue,
convention, tier, bounded context, …) are defined in the domain glossary (chapter 8). This
chapter stays light and grows only when a documentation or tooling term genuinely needs
pinning down.
