# Improvement conventions

Read by `cape:improve` as its first step. Declares this repo's **improvement mode** (see
the domain glossary, arc42 ch. 8) and any repo-specific guidance.

## Improvement mode: cape-development

This repo *is* cape. Proposing cape improvements whenever sensible — and implementing them
here now — are both options.

Follow cape's own change workflow: land changes via a feature branch and a PR into
`develop` (see [`.claude/rules/plugin-development.md`](../../.claude/rules/plugin-development.md));
board files (`docs/work/**`) and `requirements/**` commit straight to `develop`.
